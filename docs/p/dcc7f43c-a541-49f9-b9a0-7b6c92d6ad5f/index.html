Step 1: Build the Architecture to Port the HuggingFace Model

We build the architecture to port in the huggingface model.

We use huggingface model instead of original GPT2, as that one is built in tensorflow.

1.1: Architecture Overview​

Architecture is similar to GPT from previous section

Below are some tiny updates we need to make:

  1. last linear layer will use the same weights as the positional embedding layer.
  2. we will use 12 blocks (Block * 12)
  3. We will implement the multi-head-self-attention using 4-dimension tensors, instead of running tensor concatenation at the end, this will save time.
  4. to keep the variance = 1, the residual connection will divide by (sqrt(2 * num_Blocks))

Step 2: Build the Model and so that we can import the existing GPT2 weights.

import torch
from torch import nn
import torch.nn.functional as F
from dataclasses import dataclass

@dataclass
class GPTConfig:
    block_size: int = 1024 # max sequence length
    vocab_size: int = 50257 # number of tokens: 50,000 BPE merges + 256 bytes tokens + 1 <|endoftext|> token
    n_layer: int = 12 # number of layers
    n_head: int = 12 # number of heads
    n_embd: int = 768 # embedding dimension

class CausalSelfAttention(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.c_attn = nn.Linear(config.n_embd, config.n_embd * 3)
        self.c_proj = nn.Linear(config.n_embd, config.n_embd)
        self.n_embd = config.n_embd
        self.n_head = config.n_head
        self.register_buffer("bias", torch.tril(torch.ones(config.block_size, config.block_size)).view(
            1, 1, config.block_size, config.block_size
        ))

    def forward(self, x):
        B, T, C = x.size()
        qkv = self.c_attn(x)
        q, k, v = qkv.split(self.n_embd, dim=-1)
        q = q.view(B, T, self.n_head, C//self.n_head).transpose(1,2)
        k = k.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
        v = v.view(B, T, self.n_head, C // self.n_head).transpose(1, 2)
        y = F.scaled_dot_product_attention(q, k, v, is_causal=True)
        att = (q @ k.transpose(-2,-1)) / self.n_head ** 0.5
        att = att.masked_fill(self.bias[:,:,:T,:T] == 0, -torch.inf)
        att = att.softmax(dim=-1)
        y = att @ v
        y = y.transpose(1,2).contiguous().view(B, T, C)
        y = self.c_proj(y)
        return y

class MLP(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.c_fc = nn.Linear(config.n_embd, 4*config.n_embd)
        self.gelu = nn.GELU(approximate='tanh')
        self.c_proj = nn.Linear(4*config.n_embd, config.n_embd)

    def forward(self, x):
        x = self.c_fc(x)
        x = self.gelu(x)
        x = self.c_proj(x)
        return x

class Block(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.ln_1 = nn.LayerNorm(config.n_embd)
        self.attn = CausalSelfAttention(config)
        self.ln_2 = nn.LayerNorm(config.n_embd)
        self.mlp = MLP(config)

    def forward(self, x):
        x = x + self.attn(self.ln_1(x))
        x = x + self.mlp(self.ln_2(x))
        return x

class GPT(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.config = config
        self.transformer = nn.ModuleDict({
            'wte': nn.Embedding(config.vocab_size, config.n_embd),
            'wpe': nn.Embedding(config.block_size, config.n_embd),
            'h': nn.ModuleList([Block(config) for _ in range(config.n_layer)]),
            'ln_f': nn.LayerNorm(config.n_embd)
        })
        self.lm_head = nn.Linear(config.n_embd, config.vocab_size, bias=False)

    def forward(self, x):
        B, T = x.shape
        tok_emb = self.transformer.wte(x)
        pos_emb = self.transformer.wpe(torch.arange(T))
        x = tok_emb + pos_emb
        for block in self.transformer.h:
            x = block(x)
        x = self.transformer.ln_f(x)
        logits = self.lm_head(x)
        return logits

            'gpt2-xl':      dict(n_layer=48, n_head=25, n_embd=1600), # 1558M params
        }[model_type]
        config_args['vocab_size'] = 50257 # always 50257 for GPT model checkpoints
        config_args['block_size'] = 1024 # always 1024 for GPT model checkpoints
        # create a from-scratch initialized minGPT model
        config = GPTConfig(**config_args)
        model = GPT(config)
        sd = model.state_dict()
        sd_keys = sd.keys()
        sd_keys = [k for k in sd_keys if not k.endswith('.attn.bias')] # discard this mask / buffer, not a param

        # init a huggingface/transformers model
        model_hf = GPT2LMHeadModel.from_pretrained(model_type)
        sd_hf = model_hf.state_dict()

        # copy while ensuring all of the parameters are aligned and match in names and shapes
        sd_keys_hf = sd_hf.keys()
        sd_keys_hf = [k for k in sd_keys_hf if not k.endswith('.attn.masked_bias')] # ignore these, just a buffer
        sd_keys_hf = [k for k in sd_keys_hf if not k.endswith('.attn.bias')] # same, just the mask (buffer)
        transposed = ['attn.c_attn.weight', 'attn.c_proj.weight', 'mlp.c_fc.weight', 'mlp.c_proj.weight']
        # basically the openai checkpoints use a "Conv1D" module, but we only want to use a vanilla Linear
        # this means that we have to transpose these weights when we import them
        assert len(sd_keys_hf) == len(sd_keys), f"mismatched keys: {len(sd_keys_hf)} != {len(sd_keys)}"
        for k in sd_keys_hf:

                # special treatment for the Conv1D weights we need to transpose
                assert sd_hf[k].shape[::-1] == sd[k].shape
                with torch.no_grad():
                    sd[k].copy_(sd_hf[k].t())
            else:
                # vanilla copy over the other parameters
                assert sd_hf[k].shape == sd[k].shape
                with torch.no_grad():
                    sd[k].copy_(sd_hf[k])

        return model

There is only 1 difference from the GPT 1 model from previous section:

we used 4 dimension for k, q, v, this avoids concatenating tensors, as concatenating tensors takes a lot of time.

2.2. Add 3 important weight initialization:​

  1. all weights are initiated with 0.02 standard deviation
    1. this is essentially similar to the Faning initialization, but we use 0.02 to be the same as the original paper.
  2. Important: when we keep add a normal distribution of 0 mean, 1 std to an existing distribution, the variance increase, we need to counter that effect in c_proj before residual connection
  3. Most Important: the initial embedding matrix is the same as the lm_head weights, this helps us to reduce the number of parameters, in addition, if we think about the usage of those 2 weights, they essentially do the same thing.

class GPT(nn.Module):
    def __init__(self, config):

        self.config = config
        self.transformer = nn.ModuleDict({

        # weight sharing scheme
        self.transformer.wte.weight = self.lm_head.weight
        # init params
        self.apply(self._init_weights)

    def _init_weights(self, module):
        if isinstance(module, nn.Linear):
            std = 0.02
            if hasattr(module, 'RESIDUAL_CONNECTION_SCALE_INIT'):
                std *= (2 * self.config.n_layer) ** -0.5
            torch.nn.init.normal_(module.weight, mean=0.0, std=std)
            if module.bias is not None:
                torch.nn.init.zeros_(module.bias)
        elif isinstance(module, nn.Embedding):
            torch.nn.init.normal_(module.weight, mean=0.0, std=0.02)

def train_model(training_config, gpt_config):
    model = GPT(GPTConfig())
    model.train()
    optimizer = torch.optim.AdamW(params=model.parameters(), lr=3e-4)
    dataloader = ShakespeareDataLoader(training_config.batch_size, gpt_config.block_size)
    enc = tiktoken.get_encoding("gpt2")
    with open("input.txt", encoding='utf-8', mode='r') as f:
        text = f.read()
    encoded = enc.encode(text)

    for i in range(training_config.num_steps):
        start_time = time.time()
        optimizer.zero_grad()
        x, y = dataloader.next_batch(encoded)
        # with torch.autocast(device_type=device, dtype=torch.bfloat16):
        logits, loss = model(x, y)
        loss.backward()
        optimizer.step()
        if device_type == "cuda":
            torch.cuda.synchronize()  # wait for the GPU to finish work
        end_time = time.time()
        dt = (end_time - start_time) / (dataloader.B * dataloader.T) * 1000
        if i % training_config.train_interval == 0 or i == training_config.num_steps - 1:
            print(f"iteration {i=:4d} | ", f"loss={loss.item():4f} | ", f"{dt:4f} ms per token | ")

    model = model.to(device_type)
    optimizer = torch.optim.AdamW(params=model.parameters(), lr=3e-4)
    dataloader = ShakespeareDataLoader(training_config.batch_size, gpt_config.block_size)
    enc = tiktoken.get_encoding("gpt2")
    with open("input.txt", encoding='utf-8', mode='r') as f:
        text = f.read()
    encoded = enc.encode(text)

    for i in range(training_config.num_steps):
        start_time = time.time()
        optimizer.zero_grad()
        x, y = dataloader.next_batch(encoded)
        x, y = x.to(device_type), y.to(device_type)
        # with torch.autocast(device_type=device, dtype=torch.bfloat16):
        logits, loss = model(x, y)
        loss.backward()
        optimizer.step()
        if device_type == "cuda":
            torch.cuda.synchronize()  # wait for the GPU to finish work
        end_time = time.time()
        dt = (end_time - start_time) / (dataloader.B * dataloader.T) * 1000
        if i % training_config.train_interval == 0 or i == training_config.num_steps - 1:
            print(f"iteration {i=:4d} | ", f"loss={loss.item():4f} | ", f"{dt:4f} ms per token | ")
    return model

def generate_tokens(model, config):
    model.eval()
    enc = tiktoken.get_encoding('gpt2')
    tokens = enc.encode("Hello, I'm a language model, ")
    tokens = torch.tensor(tokens, dtype=torch.long)
    tokens = tokens.unsqueeze(0).repeat(4, 1)
    x = tokens.to(device_type)
    torch.manual_seed(42)
    max_length = 50
    while x.size(1) < max_length:
        with torch.no_grad():
            logits, loss = model(x[:,-config.block_size:])
            logits = logits[:, -1, :]
            probs = F.softmax(logits, dim=-1)
            topk_probs, topk_indices = torch.topk(probs, 50, dim=-1)
            ix = torch.multinomial(topk_probs, 1)
            xcol = torch.gather(topk_indices, -1, ix).to(device_type)
            x = torch.cat((x, xcol), dim=1)

    for i in range(4):
        tokens = x[i, :max_length].tolist()
        decoded = enc.decode(tokens)
        print(">", decoded)

4.3 Use FlashAttention​

FlashAttention is an algorithm that uses time (more flops) to exchange for memory efficiency.

Internally It uses an algorithm that calculates softmax online.

What it actually does is a very fast replacement to the below 4 operations

  • att = q @ k.transpose(-2,-1)
  • att.masked_fill(tril == 0, inf)
  • att = att.softmax
  • y = att @ v

i.e., the below rows

        weight = q @ k.transpose(-2, -1) / (C // num_heads) ** 0.5  # B, nh, T, T
        weight = weight.masked_fill(self.tril[:, :, :T, :T] == 0, -torch.inf)
        weight = weight.softmax(-1)
        y = weight @ v

to replace it to one line:

y = F.scaled_dot_product_attention(q, k, v, is_causal=True)

4.4 Change wierd numbers to 2**x​

we can improve speed a great deal by changing all wierd numbers to 2**x

this is because internally GPU allocate chips in kernels using 2**x (e.g., 16 chips in one unit, etc)

If we put e.g., a batch with number = 17 elements, then it is very inefficient.

It may in the end set up space allocation as, e.., 8 + 1 + 1 + 1 + 1, …, and this caused speed to be compromised.

5. three major optimization algorithms

5.1 Learning Rate Decay​

The reason why we use learning rate decay is because:

the reason for decay:

  • at the beginning, it is not actually learning much, other than understanding the basics (e.g., what tokens are not being used in the data at all, etc), so the model can tolerate a bigger learning rate.
  • We can later make learning rate smaller because that is when the model is actually learning.

the reason for warmup steps:

  • I am not sure, but I think it’s because at the very beginning the gradients are bigger, so it’s ok with a small learning rate because the weights will change to the right direction regardless.

the scale of learning rate decay:

  • max_steps: it should be the number of steps for 1 epoch.
  • warmup_steps: it should be~ max_steps//25
  • max_lr: it should be by default 3e-4
  • min_lr: max_lr * 0.1

parameters to set up in actual implementation,

  • warmup_steps = 715
  • max_steps = 19073, this is because 19,073 steps is ~1 epoch, if data is 10B tokens and batch size 0.5M tokens
import math

max_lr = 6e-4
min_lr = max_lr * 0.1
warmup_steps = 10
max_steps = 50 # 19,073 steps is ~1 epoch, if data is 10B tokens and batch size 0.5M tokens

def get_lr(it):
    # 1) linear warmup for warmup_iters steps
    if it < warmup_steps:
        return max_lr * (it+1) / warmup_steps
    # 2) if it > lr_decay_iters, return min learning rate
    if it > max_steps:
        return min_lr
    # 3) in between, use cosine decay down to min learning rate
    decay_ratio = (it - warmup_steps) / (max_steps - warmup_steps)
    assert 0 <= decay_ratio <= 1
    coeff = 0.5 * (1.0 + math.cos(math.pi * decay_ratio)) # coeff starts at 1 and goes to 0
    return min_lr + coeff * (max_lr - min_lr)

from matplotlib import pyplot as plt
plt.plot([get_lr(ele) for ele in range(55)])
plt.show()

in actual training:

lr = get_lr(it)
for parameter_group in optimizer.parameter_groups:
		parameter_group["lr"] = lr

5.2 Weight Decay​

intuition: as model gets trained, we need to pull the weights closer to 0 so that it become less affected by the extremes, and so that the gradient do not vanish.

we set up

optimizer = m.configure_optimizers(weight_decay=0.1, learning_rate=6e-4, device_type=device)

then we define the function under class GPTModel

the key is that weights that ≥2 dimension should be weight_decayed.

Note that we also need to add a fused=use_fused parameter in order for these parameters to be fused in one kernel, this increases the speed of the gradient update, and should be included as default whether or not we use weight decay.

    def configure_optimizers(self, weight_decay, learning_rate, device_type):
        param_dict = {pn: p for pn, p in self.named_parameters() if p.requires_grad}
        decay_params = [p for _, p in param_dict.items() if p.dim > 1]
        nodecay_params = [p for _, p in param_dict.items() if p.dim <= 1]
        optim_groups = [
            {'params': decay_params, 'weight_decay': weight_decay},
            {'params': nodecay_params, 'weight_decay': 0.0}
        ]
        num_decay_params = sum(p.numel() for p in decay_params)
        num_nodecay_params = sum(p.numel() for p in nodecay_params)
        print(f"num decayed parameter tensors: {len(decay_params)}, with {num_decay_params:,} parameters")
        print(f"num non-decayed parameter tensors: {len(nodecay_params)}, with {num_nodecay_params:,} parameters")
        # Create AdamW optimizer and use the fused version if it is available
        fused_available = 'fused' in inspect.signature(torch.optim.AdamW).parameters
        use_fused = fused_available and device_type == "cuda"
        print(f"using fused AdamW: {use_fused}")
        optimizer = torch.optim.AdamW(optim_groups, lr=learning_rate, betas=(0.9, 0.95), eps=1e-8, fused=use_fused)
        return optimizer

5.3 Batch Accumulation​

6. DDP

So far we are only doing 1 gpu.

We need to support doing >1 gpus.

What DDP does is that:

  • for each GPU we do the forward pass the same
  • for each backward, after the backward() is called, if we call dist.all_reduce(), we will average the weight across all the GPUs.
  • then we call step(), where the weight used in step() is the averaged weight from the all_reduce().

6.1 Step 1: Find DDP os.environ variables​

Most important os.environ parameters:

  • RANK: gpu0, gpu1, gpu2, gpu3
  • WORLD_SIZE: number of gpu (i.e., 4)
  • LOCAL_RANK: Refers to the index of the GPU on a specific machine. This is used to tell the process which GPU to use on its machine. For example:
    • On a machine with 4 GPUs, LOCAL_RANK will take values from 0 to 3.
    • This variable ensures that each process is assigned to the correct GPU on the local machine.

Let's say you're running a distributed training job with 2 machines, and each has 4 GPUs:

  • Machine 1:
    • RANK: 0, 1, 2, 3 (global ranks for the GPUs on this machine)
    • WORLD_SIZE: 8 (since there are 8 GPUs total across both machines)
    • LOCAL_RANK: 0, 1, 2, 3 (the specific GPUs on this machine)
  • Machine 2:
    • RANK: 4, 5, 6, 7 (global ranks for the GPUs on this machine)
    • WORLD_SIZE: 8
    • LOCAL_RANK: 0, 1, 2, 3 (the specific GPUs on this machine, similar to Machine 1)
ddp = os.environ.get('RANK', -1) != -1
if ddp:
    init_process_group(backend='nccl')
    ddp_rank = int(os.environ['RANK'])
    ddp_local_rank = int(os.environ['LOCAL_RANK'])
    ddp_world_size = int(os.environ['WORLD_SIZE'])
    device = f'cuda:{ddp_local_rank}'
    torch.cuda.set_device(device)
    master_process = ddp_rank == 0  # master_process is for logging
else:
    ddp_rank, ddp_local_rank, ddp_world_size, master_process = 0, 0, 1, True
    device = 'cuda' if torch.cuda.is_available() else 'cpu'

make sure to use int(), as otherwise, it is a string and cannot do the rest of the stuff.

6.2 Step 2: Use DistributedDataSampler​

Note: you must use DistributedDataSampler, instead of doing something like

        for j, data in enumerate(dataloader):
            if j % ddp_world_size != ddp_rank: continue

why:

this will cause UNEVEN distribution, and at the last round of the training, GPU7, for example, will not have any input gradient to send to other machines, and the rest of GPU will TIMEOUT while waiting for the gradient to be sent.

also, if using torchrun, then it will distribute the rest of the load to the rest of the GPUs, causing GPU overload.

6.3 Step 3: Update training process​

we need to use the model as:

model.to(device)
model = torch.compile(model)
if ddp:
		model = DDP(model, device_ids = [ddp_local_rank])

Note that now we need to save the model as

raw_model = model.module if ddp else model

this raw_model needs to be used for:

  • saving the model
  • also set up optimizers need to use raw_model.parameters()

During training:

for data loading:

  • it’s better for each data for each machine to be different,

    • e.g., we can do
    for j, data in enumerate(dataloader):
    		if j % ddp_world_size != ddp_rank: continue

If we disable DDP in the middle (e.g., like in gradient accumulation), then we need to reenable it.

To do that, after calling loss.backward()

we need to call

if ddp:
		dist.all_reduce(loss, op=dist.ReduceOp.AVG)

But we don’t need to call it otherwise.
