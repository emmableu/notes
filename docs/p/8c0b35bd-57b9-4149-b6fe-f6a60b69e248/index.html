Step 1: Define Input and Output, and Generate Output using a BiGram Language Model

我们train的目标是：

for the sentence "First Citizen:"
target input and output for block_size = 8 (meaning max input length = 8):
F -> i
Fi -> r
Fir -> s
Firs -> t
First -> ` `
First -> C
First C -> i
First Ci -> t

i.e., 我们优化parameters的方法是

如果input是 First Cit

我们优化parameters，让

x = F 时 y = i

x = Fi 时 y = r

…

x = First Ci 时 y = t

对于step 1，我们还没有掌握这种处理不同长度的input的方法，

所以，我们 x = F 时 y = i

x = Fi 时 只处理 x=i， y = r

…

x = First C 时只处理x=C y = i

x = First Ci 时只处理x=i y = t

Step 1.2: Get BiGram Language Model That can Generate Text​

It only has one layer, which is the embedding layer.

class BiGramModel(nn.Module):
    def __init__(self, vocab_size):
        super().__init__()
        self.layers = nn.Sequential(
            nn.Embedding(vocab_size, vocab_size),
            nn.Flatten(0, 1)
        )
        for layer in self.layers:
            layer.register_forward_hook(lambda layer, input, output:
                                        print(f"{layer.__class__.__name__:20} | in: {input[0].shape} | out: {output.shape}"))

    def forward(self, X):
        logits = self.layers.forward(X)
        return logits

    def generate_next(self, X, iter_i, iter_limit):
        "input is a 1-dimensional tensor, but the first dimension is always 1. e.g., 1 * 4"
        if iter_i == iter_limit:
            return X
        iter_i += 1
        logits = self.forward(X[:, -1:])  # we only need 1 input char i.e., the last one
        probs = torch.softmax(logits, dim=-1)
        ix = torch.multinomial(probs, num_samples=1)
        X = torch.concat([X, ix.view(1, 1)], dim=-1)
        return self.generate_next(X, iter_i + 1, iter_limit)

    def generate(self, X=torch.tensor([[0]]), iter_limit=100):
        x_gen = self.generate_next(X, 0, iter_limit)
        print(f"{x_gen=}")
        print(decode(x_gen.squeeze().tolist()))

m = BiGramModel(vocab_size=vocab_size)

Step 2: Build Self-Attention

2.1: self-attention trick: matrix multiplication​

Question 1: let’s say we have an X that’s [1,2,3,4], how do we update it to [1,3,6,10]?

Motivation: we want tokens to communicate to their past.

Follow-up question 2, let’s say X is [[[1,2,3,4]] * 10], how do we simultaneously update all of them to [[[1,3,6,10]] * 10] ?

直接用 [[[1,2,3,4] ]* 10] 刚刚那个中间那个matrix

用torch表示

t = torch.tensor([[1,2,3,4] for _ in range(10)], dtype=torch.float32)
mul = torch.tril(torch.ones((4,4),dtype=torch.float32)).T

res = t @ mul
print(res)
print(res.shape)

Question 3: What if X is of dimension: 10 * 4 * 3, and we want to get the sum of the 1th dimension, i,e, the dimension of 1,2,3,4?

10 * 4 * 3 @ 4* 4, this does not work, even if torch add a batch dimension so that 4 * 4 becomes 10 * 4 * 4

Let’s think about how to do it the other way around.

[[1,0,0,0],   * [1,2,3,4]  = [[1],[3],[6],[10]]
[1,1,0,0],      
[1,1,1,0],      
[1,1,1,0]]

torch.tril(torch.ones(4,4)) * 10个[1,2,3,4] 转置，就等于10个 [1,3,6,10]

4 * 4 @ 10 * 4 *3

torch会给4 * 4再加一个batch dimension， 变成 10 * 4 * 4

实际上的计算是每个batch内进行4_4 的tril和4_3的matrix的点乘

所以，最终的torch implementation是这样

t = torch.randn(10,4,3)
mul = torch.tril(torch.ones((4,4),dtype=torch.float32))

res = mul @ t
print(res.shape) # torch.Size([10, 4, 3])

Question 4: 我们现在需要的是一个weight matrix用来最终和 处理过的x进行点乘， 这个weight matrix需要满足这些条件：

  • 比如 I like green vegetable, 现在的表示是 [[I0.3,I0.5], [like0.5,like0.5],[green0.6,green0.1], [vegetable0.9,vegetable0.1]] 要变成
  [[I-I1, 0,0,0],
  [like-I0.3, like-like0.7,0,0,0]],
  [green-I0.1, green-like0.1,green-green0.8,0,0]],
  [vegetable-I0.05, vegetable-like0.15,vegetable-green0.6,vegetable-vegetable0.2]],

我们需要做的就是query 和 key 点乘，

  • query： what I want
  • key: what I have.

对于以上的例子，我们需要

[[I0.3, like0.5, green0.6, vegetable0.9],   @  [[I0.3,I0.5], 
[I0.5, like0.6, green0.1, vegetable0.1]]        [like0.5,like0.5],																						[green0.6,green0.1], 
												[vegetable0.9,vegetable0.1]]

对于这个结果，把未来未知的部分mask成inf，然后做softmax

注意 下图的-∞应该是0， 下图其实是softmax之后的结果 softmax每一行加起来应该= 1

  • Why did we use softmax at the last step for q*k?
    • 记得我们在backprop-ninja那个session推导过negative log likelihood loss对于每个元素的导数
      • 对于正确的prediction是1-p
      • 对于错误的prediction是p
    • 所以有了softmax，它可以把q*k往 越有关系，就越靠近1，越没有关系，就越靠近0 这个方向进行backpropagation

以上得到是新的weight， 再乘以处理过的x

  • 我们是否需要一直mask未来未知的部分
    • mask掉未来未知的部分的时候，我们不再关心 e.g., I-green的关系，而是只关心green-I 的关系，在transformer中这个处理是有用的，但是如果比如我们是做text analysis，e.g.，text classification，我们可以不mask掉未来的未知部分，因为我们是关心那些内容的。这个在torch的处理上就是把decoder 变成 encoder, 就会去掉mask的那一行。

torch implementation

x = torch.randn(10,4,32)
query_layer = nn.Linear(32, 2, bias=False)
key_layer = nn.Linear(32, 2, bias=False)
q = query_layer(x)
k = key_layer(x)
weight = q @ k.transpose(1,2)
tril = torch.tril(torch.ones(4,4))
weight = weight.masked_fill(tril == 0, -torch.inf) 
weight = weight.softmax(dim=-1)
value_layer = nn.Linear(32, 2, bias=False)
v = value_layer(x)
out = weight @ v
print(weight[0])
print(out.shape)

Question 5: during weight initialization, how do we make the weight less to the extreme? (remember, softmax makes bigger values even closer to 1)

answer:

we modify

weight = q @ k.transpose(1,2)

to

weight = q @ k.transpose(1,2) / head_size ** -0.5

in our case, head_size = 2.

Summary

  • I am a sentence.
  • Here’s what I am interested in (q)
  • Here’s what I have (k)
  • if I find some info interesting, here’s what I present to you (weight @ v)

def get_x_y(ix, text):
    """

def get_batch(batch_size, text):

        x_data_lst.append(x_data)
        y_data_lst.append(y_data)
    return torch.stack(x_data_lst), torch.stack(y_data_lst)




class SelfAttentionHead(nn.Module):

    def __init__(self, block_size, head_size):
        # block_size is T, head_size is C
        super().__init__()
        self.block_size = block_size
        self.head_size = head_size
        self.register_buffer(name='tril', tensor=torch.tril(torch.ones(block_size, block_size)))
        # tril is constant, not parameter, should register here.


    def forward(self, X):
        # x = torch.randn(10, 4(block_size), 32)
        B, T, C = X.shape
        query_layer = nn.Linear(C, self.head_size, bias=False)
        key_layer = nn.Linear(C, self.head_size, bias=False)
        q = query_layer(X)
        k = key_layer(X)
        weight = q @ k.transpose(1, 2)
        weight = weight.masked_fill(self.tril == 0, -torch.inf)
        weight = weight.softmax(dim=-1)
        value_layer = nn.Linear(C, self.head_size, bias=False)
        v = value_layer(X)
        out = weight @ v
        return out



class EmbeddingLayer(nn.Module):
    def __init__(self, vocab_size, block_size, n_emb):
        super().__init__()
        self.token_embedding_table = nn.Embedding(vocab_size, n_emb)
        self.position_embedding_table = nn.Embedding(block_size, n_emb)

    def forward(self, X):
        # X dim: 10, 4, n_emb=2
        B, T = X.shape
        token_emb = self.token_embedding_table(X) # 10, 4, 2
        position_emb = self.position_embedding_table(torch.arange(0, T)) # 4, 2
        return token_emb + position_emb # 10, 4, 2

Step 4: Change to Multi-Head-Attention

For the same X, go through several independent attention layers to learn their inter-connections, and in the end concatenate the results at the last dimension.

for example, if the self-attention learns a 1 * 4 * 16 matrix for [[i like green vegetables]],

if we use 4 heads, each then will learn a 1 * 4 * 4 matrix for [[i like green vegetables]],

then we concatenate the results to 1 * 4 * 16 .

update the code to add the multi-Attention module

Step 6: ResNet (VERY IMPORTANT)

This step reduced my validation loss from 3.5 to 2.5 in ONE PASS!!

Use residual connection to offset the issue of network becomes too deep and the direct, linear information from X to Y get lost through the deep network.

in short, what it does is

Y = aX + b

Z = cY + d

we change that to:

Y = aX + b + X

Z = cY + d + Y

why is this helpful:

  • addition allows gradients to distribute equally across branches
    • e.g., y = ab + ac, dy/da = b + c, addition allows the b and c to both be preserved during backward propagation.
    • This is helpful in the above case because it allows a and b to have a direct linear influence on Z, i.e., Z actually = cY + d + aX + b + X, it allows those information to flow through
    • even if we make it deeper and deeper, e.g., Y = aX + b + X, Z = cY + d + Y, Z2 = eZ + f + Z, etc, the direct relationship from final output Z2 to a and b does not get lost.

implementation

First, we need to make sure the MultiHeadAttention and the FeedForward layers indeed has a linear layer as the final layer flow out and preserve the linear information.

class MultiHeadAttention(nn.Module):
    def __init__(self, n_head, head_size):
        super().__init__()
        self.head_size = head_size
        self.heads = nn.ModuleList([
            SelfAttentionHead(head_size=head_size) for _ in range(n_head)
        ])
        **self.proj = nn.Linear(head_size * n_head, head_size * n_head)**

    def forward(self, X):
        X = torch.cat([h(X) for h in self.heads], dim=-1)
        X = self.proj(X)
        return X

class FeedForward(nn.Module):
    def __init__(self, in_features, inner_layer_features, out_features):
        super().__init__()
        self.layers = nn.Sequential(OrderedDict([
            ('linear', nn.Linear(in_features, inner_layer_features)),
            ('relu', nn.ReLU(inner_layer_features)),
            **('proj', nn.Linear(inner_layer_features, out_features)),**

        ]))
    def forward(self, X):
        return self.layers.forward(X)

We also need to make sure in the Block part there is indeed a X that flows through the information

7.1 Compare BatchNorm and LayerNorm​

Summary:​

以下用计算来表达：对于input B,T,C,

  • batchnorm取的是 B,T 的维度上的平均，最后结果
    • X[:,:,0].mean() = 0, X[:, :, 0].var() = 1
  • layernorm取的是 C 的维度上的平均，最后结果
    • X[0,0,:].mean() = 0, X[0, 0, :].var() = 1

9. Final - Network Architecture

Note: LayerNorm should not be included into the residual connection pathway

  • we want the residual connection to skip layernorm+multi-head-self-attention and, only on layernorm+feedforward only.
  • This is because we want a clean residual pathway so that X that did not get normalized get feed into the neural network.

We didn’t use the encoder part:

  • that is for calculating cross attention, which is used for machine translation.
  • We used the tril mask to run the self-attention, which is a decoder architecture.

short introduction of cross attention in machine translation:

  • e.g., if we translate from a french sentence to an english sentence
  • encoder takes the key, value from the french sentence, with the full sentence (not the tril masked one)
  • and then the cross attention uses the query to interact with the k, v.

class SelfAttentionHead(nn.Module):

    def __init__(self, head_size):
        # block_size is T, head_size is C
        super().__init__()
        self.head_size = head_size
        self.register_buffer(name='tril', tensor=torch.tril(torch.ones(block_size, block_size)))
        # tril is constant, not parameter, should register here.
        self.dropout = nn.Dropout(0.2)

    def forward(self, X):
        # x = torch.randn(10, 4(block_size), 32)
        B, T, C = X.shape
        query_layer = nn.Linear(C, self.head_size, bias=False)
        key_layer = nn.Linear(C, self.head_size, bias=False)
        q = query_layer(X)
        k = key_layer(X)
        weight = q @ k.transpose(1, 2)
        weight = weight.masked_fill(self.tril == 0, -torch.inf)
        weight = weight.softmax(dim=-1)
        weight = self.dropout(weight)
        value_layer = nn.Linear(C, self.head_size, bias=False)
        v = value_layer(X)
        out = weight @ v
        return out

    def __init__(self, vocab_size, n_emb):
        super().__init__()
        self.token_embedding_table = nn.Embedding(vocab_size, n_emb)
        self.position_embedding_table = nn.Embedding(block_size, n_emb)

    def forward(self, X):
        # X dim: 10, 4, n_emb=2
        B, T = X.shape
        token_emb = self.token_embedding_table(X)  # 10, 4, 2
        position_emb = self.position_embedding_table(torch.arange(0, T))  # 4, 2
        return token_emb + position_emb  # 10, 4, 2

class Block(nn.Module):
    def __init__(self, n_head, head_size):
        super().__init__()
        self.layers = nn.Sequential(OrderedDict([
            ('layer_norm1', nn.LayerNorm(head_size)),
            ('multi_head_attention', MultiHeadAttention(n_head, head_size=head_size // n_head)),
            ('layer_norm2', nn.LayerNorm(head_size)),
            ('feed_forward', FeedForward(head_size, head_size*4, head_size)),
        ]))

    def forward(self, X):
        for layer in self.layers:
            if layer.__class__.__name__ not in ['LayerNorm']:
                X = X + layer(X)
        return X

class GPTModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.layers = nn.Sequential(OrderedDict([
            ('embedding', EmbeddingLayer(vocab_size, n_emb)),
            ('block1', Block(n_head=4,head_size=head_size)),
            ('block2', Block(n_head=4,head_size=head_size)),
            ('block3', Block(n_head=4,head_size=head_size)),
            ('layer_norm', nn.LayerNorm(head_size)),
            ('lm_head', nn.Linear(head_size, vocab_size)),
        ])
        )
        # for layer in self.layers:
        #     layer.register_forward_hook(lambda layer, input, output:
        #                                 print(f"{layer.__class__.__name__:20} | in: {input[0].shape} | out: {output.shape}"))

    def forward(self, X):
        logits = self.layers.forward(X)
        return logits

    def generate_next(self, X, iter_i, iter_limit):
        "input is a 1-dimensional tensor, but the first dimension is always 1. e.g., 1 * 4"
        if iter_i == iter_limit:
            return X
        iter_i += 1
        # X: [1, 8]
        logits = self.forward(X[:, -block_size:])  # [1, 8, 16]
        probs = torch.softmax(logits[:, -1, :].squeeze(), dim=-1)  # [16]
        ix = torch.multinomial(probs, num_samples=1)
        X = torch.concat([X, ix.view(1, 1)], dim=-1)
        return self.generate_next(X, iter_i + 1, iter_limit)

    def generate(self, X=torch.tensor([[0 for _ in range(block_size)]]), iter_limit=1000):
        x_gen = self.generate_next(X, 0, iter_limit)
        print(decode(x_gen.squeeze().tolist()))

m = GPTModel()
train_sep_n = int(len(text) * 0.9)
train_text = text[:train_sep_n]
val_text = text[train_sep_n:]

batch_size = 64
NUM_EPOCH = 1000
optimizer = torch.optim.Adam(params=m.parameters(), lr=0.001)
loss_fn = nn.CrossEntropyLoss()
lossi = []

@torch.no_grad()
def estimate_total_loss(m, eval_iter=100):
    m.eval()
    train_loss_lst, val_loss_lst = [], []
    for _ in range(eval_iter):
        train_loss = estimate_batch_loss(m, train_text).log10().item()
        # print(f"{train_loss=}")
        val_loss = estimate_batch_loss(m, val_text).log10().item()
        # print(f"{val_loss=}")
        train_loss_lst.append(train_loss)
        val_loss_lst.append(val_loss)
    m.train()
    return sum(train_loss_lst) / eval_iter, sum(val_loss_lst) / eval_iter

def estimate_batch_loss(m, text):
    X, Y = get_batch(batch_size=batch_size, text=text)
    logits = m.forward(X).flatten(0, 1)
    loss = loss_fn(input=logits, target=Y.flatten(0, 1))
    return loss

train_lossi, val_lossi = [], []

def train():
    m.train()
    for i in tqdm(range(NUM_EPOCH)):
        optimizer.zero_grad()
        loss = estimate_batch_loss(m, train_text)
        loss.backward()
        if i % 100 == 0:
            cur_train_loss, cur_val_loss = estimate_total_loss(m)
            train_lossi.append(cur_train_loss)
            val_lossi.append(cur_val_loss)
        optimizer.step()

train()

Appendix

Other tokenizers:​

  • Google uses SentencePiece. It is a sub-word tokenizer (between 1 char and 1 word)
  • OpenAI uses tiktoken, it uses BPE (byte-pair encoding) that’s similar to sub-word tokenizer.
  • e.g., hii there has 3 tokens.

nn.Flatten & nn.Unflatten​

nn.Flatten: put a >2 dimension data to 2 dimension.​

  • default
    • is m = nn.Flatten(start_dim=1,end_dim=-1),
t = torch.randn((1,2,3,4))
m = nn.Flatten()
out = m(t)
print(out.shape) # (1, 24)

  • if I do

    • m = nn.Flatten(0,1)
    • then out shape will be (2, 12) → (explicitly) flatten first dimension, and flatten whatever is rest as the second dimension

  • if I do

    • m = nn.Flaten(2,3)
    • then out will be (2, 12) too →(explicitly) flatten second dimension, and flatten whatever is rest as the first dimension

nn.Unflatten: put a 2 dimension data to > 2 dimension​

need to specify: which dimension, make it into what

t = torch.randn(24, 63)
m = nn.Unflatten(dim=0, unflattened_size=(1, 4, 6))
out = m(t)
print(out.shape) # (1, 4, 6, 63)
t = torch.randn(24, 63)
m = nn.Unflatten(dim=1, unflattened_size=(9, 7))
out = m(t) 
print(out.shape) # (24, 9, 7)
