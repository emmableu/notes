第2章 多臂混混机问题 (Multi-Arm Bandits)

基本概念​

强化学习与其他类型学习最重要的区别是：它使用“评估式反馈”而非“指示式反馈”。

  • 评估式反馈：只告诉你选的这个动作“好不好”，但不告诉你是不是最好的
  • 指示式反馈：明确指出哪个动作是正确的，与是否执行无关

2.1 nnn 臂混混机问题

  • 每次从 nnn 个动作中选择一个
  • 每个动作对应一个固定但未知的奖励分布
  • 目标是最大化未来若干步内的累计期望奖励，例如 1000 步内

请考虑以下学习问题：
你反复面临从多个不同选项（或动作）中进行选择的问题。每次选择后，你会收到一个根据所选动作的固定概率分布生成的数值奖励。
你的目标是：最大化某一时间段内的期望总奖励，例如在 1000 次动作选择或时间步内。

这就是著名的 n 臂老虎机问题（n-armed bandit problem） 的原始形式。其名称来源于老虎机——又称为“单臂强盗（one-armed bandit）”，不过它拥有 n 个拉杆，而非一个。

每一个动作选择就像是在老虎机上拉一次拉杆，而奖励就是命中“头奖”后的收益。

通过多次重复的选择，你希望通过集中选择某些拉杆来最大化你的盈利。另一种类比是医生为多位重病患者选择治疗方案，每次选择一个治疗手段，而每次的“奖励”就是该患者的康复程度。今天，“n 臂老虎机问题”这个术语通常泛指此类问题的推广形式，但在本书中我们专指此种形式。

在我们的 n 臂老虎机问题中，每个动作都有一个期望奖励（mean reward），假设我们将某一动作的期望奖励称作该动作的“值（value）”。

若你知道每个动作的值，那么解决该问题是非常简单的：你总是选择值最大的动作即可。
但现实中我们并不知道各动作的值，我们只能逐步估计它们。

因此，若你在某一时刻估计的动作值中，至少有一个的估值最大，那么选择它即是一个贪婪动作（greedy action）。如果你选择这样的动作，即说明你正在利用（exploiting） 你对当前动作值的已有知识。

反之，若你选择了一个非贪婪的动作，即说明你正在探索（exploring），以尝试获得对该动作值的更好估计。利用是从当前角度来看最优的选择，但探索可能带来更大的长期收益。
举例来说：假设贪婪动作的值是已知且确定的，而另一些动作的估值接近，但不确定性很大。那么至少有一个非贪婪动作可能实际上比贪婪动作更优，但你并不知道哪一个。因此如果你有足够多的时间步用于做选择，那么探索这些不确定的动作并找出更优的选择可能更有利。

短期来看探索带来的奖励可能较低，但长期来看，由于发现了更优的选择，你可能获得更大的收益。
由于无法在单次选择中既探索又利用，因此“探索 vs 利用”的冲突成为强化学习中一个核心问题。

在某些特定情况下，是否该探索或利用依赖于估值的准确性、不确定性、剩余时间步数等因素的复杂组合。
虽然存在很多方法平衡探索与利用，例如对 n 臂老虎机问题和其推广问题的数学形式进行优化，但大多数方法假设环境是平稳的（stationary） 且事先可知的，这在现实世界中往往不成立。

本书不会从过于复杂的角度来研究如何平衡探索与利用。我们甚至不会过分担心这种平衡，而是重点介绍几种简单有效的策略。这些方法远优于只利用、永远选择当前最优动作的方法。探索与利用的平衡是强化学习的典型挑战，而 n 臂老虎机的简单设置使我们能在清晰的框架中展现这一挑战。

2.2 动作价值方法（Action-Value Methods）与 ε-greedy 策略详解

我们接下来要更深入地研究一些估计动作值（value） 的简单方法，并使用这些估计值来指导动作选择决策。

在本章中，我们用 Qt(a)Q_t(a)Qt​(a) 来表示第 ttt 步时对动作 aaa 的值（即平均奖励）的估计。
真实动作值是执行该动作时获得的期望奖励，而估计值就是对该期望的近似。

一种自然的估计方法是取每次选择该动作时获得的实际奖励的平均值。

换句话说，若在第 ttt 步之前，动作 aaa 被选择了 Nt(a)N_t(a)Nt​(a) 次，获得了奖励 R1,R2,...,RNt(a)R_1, R_2, ..., R_{N_t(a)}R1​,R2​,...,RNt​(a)​，那么对动作 aaa 的值估计为：

Qt(a)=R1+R2+⋯+RNt(a)Nt(a)(2.1)Q_t(a) = \frac{R_1 + R_2 + \cdots + R_{N_t(a)}}{N_t(a)} \tag{2.1}Qt​(a)=Nt​(a)R1​+R2​+⋯+RNt​(a)​​(2.1)

✅ 等价递推更新公式​

在实际实现中，我们不需要每次重新遍历所有过去的奖励 RiR_iRi​ 来计算平均值，而可以使用一个在线更新公式（incremental update）：

Q[action] += (reward - Q[action]) / N[action]

这是 式 (2.1) 的递推更新形式。它允许我们在每次收到新奖励后，仅使用当前估计值和新奖励来更新，不需要保存所有历史数据。

📚 使用 Sample Average 的动作选择方法

如果 Nt(a)=0N_t(a) = 0Nt​(a)=0，那么我们可以定义 Q1(a)Q_1(a)Q1​(a) 为某个默认值，例如 Q1(a)=0Q_1(a) = 0Q1​(a)=0。随着 Nt(a)→∞N_t(a) \to \inftyNt​(a)→∞，根据大数定律，Qt(a)Q_t(a)Qt​(a) 会收敛到 q(a)q(a)q(a)。我们将这种方法称为 sample-average method（样本均值法），因为每次估计值是该动作相关奖励的简单平均。

当然这只是众多估计动作值的方法之一，并非最优方法。尽管如此，我们暂且从这个简单方法开始，接下来讨论如何使用这些估计值来选择动作。

🤖 贪婪与 ε-贪婪动作选择策略

最简单的动作选择策略是：选择当前具有最高估计值的动作（或之一）。也就是在第 ttt 步选择：

At=arg⁡max⁡aQt(a)(2.2)A_t = \arg\max_a Q_t(a) \tag{2.2}At​=argamax​Qt​(a)(2.2)

其中 arg⁡max⁡a\arg\max_aargmaxa​ 表示在所有动作 aaa 中选取使得表达式取最大值的动作（如遇平局则随机选择）。该策略称为 贪婪策略（greedy action selection）。

这种策略完全基于当前知识做出最大化预期奖励的选择，没有探索其他看似较差但可能实际更好的动作。

一个替代策略是使用 ε-贪婪方法（ε-greedy）： • 在绝大多数时间（概率 1−ε1 - \varepsilon1−ε）选择当前估计值最高的动作 • 在少数时间（概率 ε\varepsilonε）随机从所有动作中选择一个

这种方式会确保每个动作最终都会被采样无穷次，从而满足 Qt(a)→q(a)Q_t(a) \to q(a)Qt​(a)→q(a)，即估计值最终会收敛到真实值。

这意味着选择最优动作的概率也会收敛到接近 1−ε1 - \varepsilon1−ε，即接近确定性。

这些只是渐近性质，实际效果还需实证分析。

class Bandit:
    def __init__(self, epsilon):
        self.q_star_a = torch.randn((10,))
        self.optimal_action = self.q_star_a.argmax().item()
        self.cur_t = 0
        self.n_t_a = torch.zeros((10,))
        self.big_q_a = torch.zeros((10,))
        self.epsilon = epsilon

    def select(self):
        self.cur_t += 1
        if torch.rand((1,)).item() < self.epsilon:
            a = torch.randint(0, self.q_star_a.size(0), (1,)).item()
        else:
            a = self.big_q_a.argmax().item()
        self.n_t_a[a] += 1
        return a

    def calc_r_a(self, a):
        return (self.q_star_a[a] + torch.randn((1,))).item()

    def calc_big_q_a(self, a, r_a):
        self.big_q_a[a] += (r_a - self.big_q_a[a]) / self.n_t_a[a]

import torch
import matplotlib.pyplot as plt


class Bandit:
    def __init__(self, epsilon):
        self.q_star_a = torch.randn((10,))
        self.optimal_action = self.q_star_a.argmax().item()
        self.cur_t = 0
        self.n_t_a = torch.zeros((10,))
        self.big_q_a = torch.zeros((10,))
        self.epsilon = epsilon

    def select(self):
        self.cur_t += 1
        if torch.rand((1,)).item() < self.epsilon:
            a = torch.randint(0, self.q_star_a.size(0), (1,)).item()
        else:
            a = self.big_q_a.argmax().item()
        self.n_t_a[a] += 1
        return a

    def calc_r_a(self, a):
        return (self.q_star_a[a] + torch.randn((1,))).item()

    def calc_big_q_a(self, a, r_a):
        self.big_q_a[a] += (r_a - self.big_q_a[a]) / self.n_t_a[a]

    def run_bandit(self, ts=5):
        reward = []
        optimal_action = []
        for _ in range(ts):
            a = self.select()
            r_a = self.calc_r_a(a)
            self.calc_big_q_a(a, r_a)
            reward.append(r_a)
            optimal_action.append(float(a == self.optimal_action))

        reward = torch.tensor(reward)
        optimal_action = torch.tensor(optimal_action)

        return reward, optimal_action

def one_round_experiment(epsilon):
    reward_lst = []
    optimal_action_lst = []
    for _ in range(2000):
        b = Bandit(epsilon)
        reward, optimal_action = b.run_bandit(ts=1000)
        reward_lst.append(reward)
        optimal_action_lst.append(optimal_action)

    avg_reward_tensor = torch.stack(reward_lst) # (1000) => (2000, 1000)
    perc_optimal_action_tensor = torch.stack(optimal_action_lst) # (1000) => (2000, 1000)
    return avg_reward_tensor.mean(0), perc_optimal_action_tensor.mean(0)

2.4 跟踪非平稳问题（Tracking a Nonstationary Problem）

我们之前讨论的平均方法，适用于平稳环境，也就是奖励不会随时间变化的情况。但在实际中，我们经常遇到非平稳的强化学习问题，例如 bandit 的奖励分布可能随时间变化。

在这种情况下，我们通常会希望更重视近期的奖励而不是很久以前的。最常用的方法是使用一个固定的步长参数（step-size parameter）来更新平均值。

例如，第 kkk 次对某动作的动作估计值更新公式可以写作：

Qk+1=Qk+α[Rk−Qk](2.5)Q_{k+1} = Q_k + \alpha \left[ R_k - Q_k \right] \tag{2.5}Qk+1​=Qk​+α[Rk​−Qk​](2.5)

其中 α∈(0,1]\alpha \in (0, 1]α∈(0,1] 是一个常数步长参数。这使得 Qk+1Q_{k+1}Qk+1​ 是一个带有权重的过去奖励加权平均值。

关于收敛的数学条件​

有时我们希望步长 αk\alpha_kαk​ 随时间变化，即每一步使用不同的 αk(a)\alpha_k(a)αk​(a)，例如当动作 aaa 被第 kkk 次选中时，使用步长 αk(a)\alpha_k(a)αk​(a)。如果我们取 αk(a)=1k\alpha_k(a) = \frac{1}{k}αk​(a)=k1​，就得到样本平均法，它是收敛的。

但要想保证收敛，步长序列必须满足以下条件（随机近似理论）：

∑k=1∞αk(a)=∞且∑k=1∞αk2(a)<∞(2.7)\sum_{k=1}^{\infty} \alpha_k(a) = \infty \quad \text{且} \quad \sum_{k=1}^{\infty} \alpha_k^2(a) < \infty \tag{2.7}k=1∑∞​αk​(a)=∞且k=1∑∞​αk2​(a)<∞(2.7)

第一个条件保证最终能克服初始偏差和噪声；第二个条件保证最终收敛。

实际应用中的选择​

虽然 αk=1k\alpha_k = \frac{1}{k}αk​=k1​ 的样本平均法满足上述条件，但固定步长 α\alphaα 不满足第二个条件，因此在理论上不收敛。但我们仍然倾向于使用常数步长法，因为它可以持续响应最新奖励，非常适合非平稳问题。

事实上，非平稳环境在实际中非常常见，因此固定步长法在实际应用中更常见。尽管满足 (2.7) 的步长序列在理论研究中使用广泛，但在实践中较少使用。

2.5 乐观初始值（Optimistic Initial Values）

我们到目前为止讨论的所有方法，在某种程度上都依赖于动作价值估计的初始值 Q1(a)Q_1(a)Q1​(a)。在统计学中，这些方法被认为是有偏的（biased），因为它们受初始估计的影响。

虽然这种偏差在所有动作都被尝试一次后会消失，但在使用常数步长 α\alphaα 的方法中，这种偏差是永久存在的，尽管会随时间逐渐减弱（如公式 2.6 所示）。实际上，这种偏差通常不是问题，有时甚至是非常有用的。问题在于，初始估计变成了需要人为设定的一组参数（例如都设为 0）。但好处是，它为用户提供了一种途径，向系统注入对奖励水平的先验知识。

初始值也可以鼓励探索​

我们也可以将初始估计用作鼓励探索的一种简单方法。假设我们将 Q1(a)Q_1(a)Q1​(a) 设置为一个较大的值，例如 5，而不是默认的 0（如 10 臂 bandit 中那样）。回忆一下，这里 q(a)q(a)q(a) 是从均值为 0、方差为 1 的正态分布中抽取的 —— 所以设为 5 是有意乐观的（optimistic）。

这个乐观估计鼓励动作方法进行探索。无论哪个动作一开始被选中，得到的奖励都小于估计值，因此 agent 会“失望”，然后尝试其他动作。结果就是，所有动作都被尝试多次，直到估计值收敛为止。

即使是贪婪方法也可以在所有动作都被尝试至少一次前获得不少探索。

图示与分析​

Figure 2.2 展示了 10 臂 bandit 问题中，两种贪婪方法的对比：

  • 乐观贪婪法（optimistic greedy）：Q1(a)=+5Q_1(a)=+5Q1​(a)=+5，ϵ=0\epsilon=0ϵ=0
  • 现实贪婪法（realistic ε-greedy）：Q1(a)=0Q_1(a)=0Q1​(a)=0，ϵ=0.1\epsilon=0.1ϵ=0.1

结果表明，乐观方法在初期效果更好，因为它鼓励了探索；随着时间推移，现实方法最终表现更优，但探索来得更慢。

这种技巧被称为乐观初始值（optimistic initial values），是一个非常有效的探索方式 —— 虽然只是一个简单技巧，但其效果却非常突出。

然而，它从通用性角度看仍不如 ε-greedy，更适合作为启发性方法在有些问题中使用。

2.6 置信上界动作选择（Upper-Confidence-Bound Action Selection）

我们需要探索（exploration），因为当前对动作价值的估计仍然不确定。贪婪策略总是选择当前看来最好的动作，但其他动作也有可能更优。ε-greedy 策略虽然能尝试非贪婪动作，但这种尝试是随机的，不考虑哪些动作更可能有潜力或更不确定。

更好的方法是，在选择动作时考虑两个因素：

  1. 当前动作价值的估计值；
  2. 该估计值的不确定性。

一种有效的选择方式是使用置信上界（UCB）算法，其动作选择策略为：

At=arg⁡max⁡a[Qt(a)+cln⁡tNt(a)](2.8)A_t = \arg\max_a \left[ Q_t(a) + c \sqrt{\frac{\ln t}{N_t(a)}} \right] \tag{2.8}At​=argamax​[Qt​(a)+cNt​(a)lnt​​](2.8)

其中：

  • ln⁡t\ln tlnt 是时间步 ttt 的自然对数（e≈2.71828e \approx 2.71828e≈2.71828 是自然对数的底数）；
  • Nt(a)N_t(a)Nt​(a) 是动作 aaa 到目前为止被选中的次数；
  • c>0c > 0c>0 是一个超参数，用来控制探索的程度；
  • 如果 Nt(a)=0N_t(a)=0Nt​(a)=0，那么该动作 aaa 被视为当前最大化目标（即必须尝试）。

置信上界的含义和作用​

UCB 方法的思想是：ln⁡t/Nt(a)\sqrt{\ln t / N_t(a)}lnt/Nt​(a)​ 项衡量了估计值的不确定性或方差。

  • 这个“上界”项的作用，是为动作价值设定一个可能的上限，ccc 决定了这个置信区间的宽度。
  • 每次动作 aaa 被选中时，Nt(a)N_t(a)Nt​(a) 增加，因此该项减小（代表我们对该动作更确定）。
  • 相反，如果某个动作长时间没有被选中，那么随着 ttt 增大，ln⁡t\ln tlnt 也会增大，使得该动作的置信上界增加，从而最终会被重新选中。

这种机制保证了所有动作最终都会被尝试，探索是自适应而非随机的，比 ε-greedy 更精细。

图示说明与效果​

Figure 2.3 展示了在 10 臂 bandit 测试中的 UCB（c=2c=2c=2）与 ε-greedy（ε=0.1ε=0.1ε=0.1）的对比。可以看出：

  • UCB 一般性能更好，尤其在中后期；
  • 初始前 nnn 步（每个动作都未尝试过时），UCB 选择是随机的；
  • 当 c=1c=1c=1 时，表现更好但不会像 c=2c=2c=2 那样在第 11 步产生显著的性能峰值。

这种峰值可以通过理解置信上界机制解释：一旦所有动作都被尝试一次，UCB 开始有信息性地选择最优动作。

UCB 的应用挑战​

虽然 UCB 在 10 臂 bandit 问题中表现出色，但将其拓展到更广义的强化学习任务（如马尔可夫决策过程）是有挑战的：

  • 一是非平稳性问题（见 2.4 节），需要更复杂的机制；
  • 二是在大状态空间或需要函数逼近（如神经网络）的环境中，UCB 的置信区间难以定义或估计。

因此，在实际复杂 RL 任务中，当前还没有成熟可行的通用 UCB 方法。但它作为理论启发和基础方法，仍然非常重要。

2.7 梯度 Bandit 方法（Gradient Bandits）

前面我们介绍的方法都是对动作价值进行估计，并使用这些估计来选择动作。这是一种常见且有效的策略，但并不是唯一的方法。

本节我们介绍另一种方法，它为每个动作 aaa 学习一个数值偏好值 Ht(a)H_t(a)Ht​(a)。偏好值越大，表示越倾向选择该动作。但偏好值本身不表示奖励的多少，它只用于控制选择概率。

Softmax 概率选择​

我们使用 softmax 分布来将偏好值转换为概率（也称为 Gibbs 或 Boltzmann 分布）：

Pr(At=a)=eHt(a)∑b=1neHt(b)=πt(a)(2.9)\text{Pr}(A_t = a) = \frac{e^{H_t(a)}}{\sum_{b=1}^n e^{H_t(b)}} = \pi_t(a) \tag{2.9}Pr(At​=a)=∑b=1n​eHt​(b)eHt​(a)​=πt​(a)(2.9)

注意：

  • 初始时所有 H1(a)=0H_1(a) = 0H1​(a)=0，所以每个动作都有相等概率；
  • πt(a)\pi_t(a)πt​(a) 表示第 ttt 步时选择动作 aaa 的概率。

偏好的更新公式​

每一步选择 AtA_tAt​ 并获得奖励 RtR_tRt​ 后，使用偏好值 HHH 的更新如下：

Ht+1(a)=Ht(a)+α(Rt−Rˉt)[1a=At−πt(a)]H_{t+1}(a) = H_t(a) + \alpha (R_t - \bar{R}_t) \left[ \mathbb{1}_{a=A_t} - \pi_t(a) \right]Ht+1​(a)=Ht​(a)+α(Rt​−Rˉt​)[1a=At​​−πt​(a)]

其中： • 1a=At\mathbb{1}_{a=A_t}1a=At​​ 表示 如果 a=Ata=A_ta=At​ 则值为 1，否则为 0 • πt(a)\pi_t(a)πt​(a) 是 softmax 输出的概率 • (Rt−Rˉt)(R_t - \bar{R}_t)(Rt​−Rˉt​) 是与平均奖励的对比，起到 baseline 的作用
