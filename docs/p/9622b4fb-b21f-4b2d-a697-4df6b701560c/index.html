偏度（Skewness）​

  • 定义：衡量分布的对称性。
  • 公式： Skewness=E[(X−μ)3]σ3\text{Skewness} = \frac{E[(X - \mu)^3]}{\sigma^3}Skewness=σ3E[(X−μ)3]​ 其中 (\mu) 是均值，(\sigma) 是标准差。
  • 取值解释：
    • Skewness = 0：分布完全对称。
    • Skewness > 0：分布右偏（右尾更长）。
    • Skewness < 0：分布左偏（左尾更长）。

在正态分布中：

Skewness=0\text{Skewness} = 0Skewness=0

峰度（Kurtosis, pandas 定义）​

  • 定义：衡量分布的尖峭程度及尾部厚度。
    在 pandas 中，Series.kurt() 返回的是 超额峰度 (Excess Kurtosis)：

    Excess Kurtosis=E[(X−μ)4]σ4−3\text{Excess Kurtosis} = \frac{E[(X - \mu)^4]}{\sigma^4} - 3Excess Kurtosis=σ4E[(X−μ)4]​−3

  • 取值解释：

    • 正态分布的超额峰度 = 0。
    • 超额峰度 > 0：高峰态（Leptokurtic），比正态更尖、更厚尾。
    • 超额峰度 < 0：低峰态（Platykurtic），比正态更平、更轻尾。

在正态分布中 (pandas 定义)：

Kurtosis=0\text{Kurtosis} = 0Kurtosis=0

Python 代码示例

import numpy as np
# 生成标准正态分布样本
np.random.seed(42)
data = np.random.normal(loc=0, scale=1, size=100000)

# 计算偏度和峰度
skewness = data.skew()
kurt = data.kurt()  # fisher=False 表示计算普通峰度（正态=3）

print("Skewness:", skewness)
print("Kurtosis:", kurt)

Q4.1.1：为什么不能只把训练数据简单分为 train/test 两份？ 👉 思路：单次划分可能过拟合特定子集，不能代表泛化性能。

4.2 正则化回归

Ridge 回归

Q4.2.1：Ridge 回归的目标函数是什么？为什么能缓解过拟合？ 👉 思路： J(θ)=12m∑(hθ(x(i))−y(i))2+λ2m∑θj2J(\theta) = \frac{1}{2m} \sum (h_\theta(x^{(i)}) - y^{(i)})^2 + \frac{\lambda}{2m} \sum \theta_j^2J(θ)=2m1​∑(hθ​(x(i))−y(i))2+2mλ​∑θj2​ L2 惩罚项抑制参数过大。

Q4.2.2：如何选择最佳正则化参数 λ？ 👉 思路：用交叉验证，在一组候选 λ 上跑模型，找 RMSLE 最低点。 👉 输出/检查：绘制 λ vs RMSLE 曲线，报告最佳 λ 和对应 RMSLE。

Lasso 回归

Q4.2.3：Lasso 回归与 Ridge 的主要区别是什么？ 👉 思路：Lasso 惩罚项为 |\theta_j|，能把部分系数直接压到 0，从而实现特征选择。

Q4.2.4：在本数据上，Lasso 的最佳 λ 大约是多少？ 👉 输出/检查：通过交叉验证，找到 RMSLE 最低点。

XGBoost

Q4.3.1：XGBoost 相比线性模型的优势在哪里？ 👉 思路：能捕捉非线性和高阶交互；支持正则化；速度快。

Q4.3.2：XGBoost 调参时，哪些是关键参数？ 👉 示例：n_estimators, max_depth, min_child_weight, gamma, subsample, colsample_bytree, reg_alpha, reg_lambda, learning_rate。 👉 输出/检查：在调好的参数下，记录 RMSLE。

LightGBM

Q4.3.3：LightGBM 与 XGBoost 的核心区别是什么？ 👉 思路：LightGBM 使用基于直方图的算法，支持 leaf-wise 生长，训练更快。

Q4.3.4：LightGBM 的关键参数有哪些？ 👉 示例：num_leaves, min_data_in_leaf, max_depth, max_bin, bagging_fraction, feature_fraction。 👉 输出/检查：对比 RMSLE，通常 LGBM 会比 XGB 稍好。

4.4 模型融合

Q4.4.1：为什么要做模型融合（Averaging/Stacking）？ 👉 思路：不同模型在不同数据子集上表现不同，融合能减少方差，提高稳定性。

Q4.4.2：我们如何实现一个简单的 Averaging 模型？ 👉 思路：训练多个基模型（Ridge, Lasso, XGB, LGBM），预测时取平均。

Q4.4.3：融合后的 RMSLE 相比单模型提升多少？ 👉 输出/检查：RMSLE < 各单模型；说明泛化性能更好。

4.5 最终预测与提交

Q4.5.1：为什么在预测输出前要用 np.exp(predictions) - 1 还原？ 👉 思路：因为我们在建模时用了 log(1+y)，要反变换回原始价格。

Q4.5.2：提交文件格式应是什么样子？ 👉 思路：两列：Id 和 SalePrice。 👉 输出/检查：前 5 行打印，确认格式符合 Kaggle 要求。

  1. 结论

Q5.1：整个流程中，你觉得提升最大的环节是什么？ 👉 思路：特征工程（构造 totalSqFeet、totalBathroom 等）和模型融合。

Q5.2：如果你要进一步提升成绩，你会尝试哪些方法？ 👉 示例：Stacking 更复杂的模型；特征选择；外部数据引入（宏观经济指标）。

LightGBM 原理问答

⸻

  1. 决策树基础

Q1.1：决策树是怎么划分数据的？ 👉 思路：基于某个特征的阈值，把样本划分成更“纯”的子集。常用指标有 均方误差（回归） 或 信息增益/基尼指数（分类）。

Q1.2：树的深度和叶子数对模型有什么影响？ 👉 思路：更深 → 拟合更精细，风险是过拟合；太浅 → 偏差大，欠拟合。

⸻

  1. Boosting 框架

Q2.1：什么是 Boosting？ 👉 思路：逐步训练弱学习器（通常是浅树），每一步纠正前一步的误差，最后加权组合。

Q2.2：Boosting 与 Bagging（如随机森林）的区别是什么？ 👉 思路： • Bagging → 并行，降低方差，树之间独立。 • Boosting → 串行，逐步拟合残差，降低偏差。

Q2.3：为什么梯度提升树（GBDT）比简单的 AdaBoost 更强？ 👉 思路：GBDT 用梯度下降的思想来拟合残差，可处理任意可微损失函数（平方误差、逻辑损失等）。

  1. LightGBM 的关键机制

Q3.1：LightGBM 与 XGBoost 最大的不同是什么？ 👉 思路：LightGBM 使用 基于直方图 (Histogram-based) 的分裂方法，内存更省，训练更快。

Q3.2：LightGBM 为什么快？ 👉 关键点： 1. 直方图优化：连续特征先离散成桶（bin），减少计算量。 2. Leaf-wise 分裂：不是逐层扩展，而是每次优先扩展“增益最大的叶子”。 3. 支持大规模数据：优化了并行与内存使用。

Q3.3：什么是 Leaf-wise 分裂？ 👉 思路： • 传统 GBDT 是 Level-wise（逐层分裂）。 • LightGBM 是 Leaf-wise，每次选择增益最大的叶子继续分裂。 👉 优势：更快降低损失。 👉 风险：容易过拟合，需要通过 max_depth 等参数约束。

Q3.4：LightGBM 中常见的正则化手段有哪些？ 👉 示例： • min_data_in_leaf（叶子最小样本数） • max_depth（最大树深度） • feature_fraction（随机特征采样） • bagging_fraction（随机样本采样） • lambda_l1, lambda_l2（L1/L2 正则）

⸻

  1. LightGBM 的应用与解释

Q4.1：LightGBM 如何处理类别型特征？ 👉 思路：它原生支持类别特征（categorical_feature 参数），可以用最优分裂而不是 One-hot，全局更高效。

Q4.2：如果 LightGBM 出现过拟合，常见的解决方案有哪些？ 👉 思路： • 限制 num_leaves（叶子数）； • 增大 min_data_in_leaf； • 限制 max_depth； • 增加 bagging_fraction、feature_fraction； • 调小 learning_rate，增大 n_estimators。

Q4.3：如何可视化 LightGBM 的决策过程？ 👉 思路： • plot_importance() → 特征重要性。 • plot_tree() 或导出 Booster → 树结构可视化。 • shap.TreeExplainer → 单样本决策路径解释。

⸻

  1. 面试延伸问题

Q5.1：为什么 LightGBM 在大数据集上比 XGBoost 更快？ 👉 回答关键词：直方图近似、leaf-wise 策略、内存优化。

Q5.2：Leaf-wise 虽然快，但容易过拟合，如何在面试中解释调参思路？ 👉 回答关键词：加正则约束（num_leaves, min_data_in_leaf, max_depth），结合交叉验证选择。

模型解释 (Model Interpretability) — 问答题

  1. 特征重要性 (Feature Importance)

Q1.1：树模型（如 LightGBM、XGBoost）里，如何定义特征重要性？ 👉 思路：常见方法包括： • Split importance（特征被分裂的次数） • Gain importance（使用该特征带来的损失函数下降） • Permutation importance（打乱该特征后性能下降程度）

Q1.2：为什么要小心解释特征重要性？ 👉 思路： • 重要性只表示模型在训练时“用过”该特征，不代表因果关系。 • 与其他高度相关的特征（多重共线性）会“分摊”重要性。

Q1.3：如何可视化 LightGBM 的特征重要性？ 👉 思路：使用 plot_importance 或者手动绘制条形图。 👉 输出/检查：Top 20 特征的重要性排序。

  1. SHAP (SHapley Additive exPlanations)

Q2.1：SHAP 值的核心思想是什么？ 👉 思路：基于博弈论中的 Shapley Value，计算每个特征在“预测贡献”中的边际作用。

Q2.2：为什么 SHAP 比简单的特征重要性更强大？ 👉 思路： • 提供 全局解释（哪些特征整体最重要）； • 也提供 局部解释（为什么这个样本的预测是 xxx）。

Q2.3：如何解释 SHAP summary plot？ 👉 思路： • y 轴：特征名（按重要性排序）。 • x 轴：SHAP 值大小（对预测的正负贡献）。 • 颜色：特征取值大小（红色高、蓝色低）。

Q2.4：如何用 SHAP 分析单个样本？ 👉 思路：用 shap.force_plot 或 shap.waterfall_plot，能展示预测值是如何从基准值逐步被各个特征推高/推低的。

  1. 面试延伸问题

Q3.1：在实际业务中，为什么“模型解释”很重要？ 👉 思路： • 业务可信度：告诉非技术方“模型是如何判断的”。 • 合规需求：某些场景（金融、医疗）必须能解释决策。 • 模型调试：帮助发现异常特征或错误信号。

Q3.2：如果 OverallQual 和 GrLivArea 高度相关，SHAP 会如何分配贡献？ 👉 思路：SHAP 会在多种特征排列顺序下平均分摊贡献，因此结果比简单的特征重要性更公平。
