示例 1：

输入: n = 3, k = 2, row = 0, column = 0 输出: 0.0625 解释: 有两步(到(1,2)，(2,1))可以让骑士留在棋盘上。 在每一个位置上，也有两种移动可以让骑士留在棋盘上。 骑士留在棋盘上的总概率是0.0625。

示例 2：

输入: n = 1, k = 0, row = 0, column = 0 输出: 1.00000

提示:

  • 1 <= n <= 25
  • 0 <= k <= 100
  • 0 <= row, column <= n - 1

LeetCode 688: Knight Probability in Chessboard —— MDP 框架提示

1. 定义 MDP 五元组​

  • 状态空间 S

    • 三个维度 (i, j, k)
      • i, j = 骑士所在棋盘位置
      • k = 剩余的步数

  • 动作空间 A

    • 骑士有 8 种可能的走法 (固定集合)。

  • 状态转移概率 P(s' | s, a)

    • 从 (i, j, k) 经过动作 a 转移到 (i', j', k-1)。
    • 每个动作的概率 = 1/8（均匀分布）。

  • 奖励函数 R(s,a,s')

    • 不是每一步即时奖励，而是 是否出界。
    • 可以理解为：如果出界，最终 return = 0；如果存活，最终 return = 1。当前没有奖励函数

  • 折扣因子 γ

    • 这里没有无限远未来，问题是有限 horizon (最多 k 步)。
    • 等价于 γ = 1。
  1. 状态价值函数 = V • 英文是 Value function of a state。 • 所以记作 V(s)。 • 它表示：从某个状态 s 出发，按照某个策略 π 走下去，未来能得到的 期望回报。 • 直观理解：V = Value of being in a state。

例子： 在 688 骑士题里， V(i,j,k)={从位置(i,j)，剩余k步时，存活下来的概率}V(i,j,k) = \text\{从位置 (i,j)，剩余 k 步时，存活下来的概率\}V(i,j,k)={从位置(i,j)，剩余k步时，存活下来的概率}

  1. 动作价值函数 = Q • 英文是 Quality function（或者说 Action-Value function）。 • 所以记作 Q(s,a)。 • 它表示：在状态 s 下，先采取动作 a，然后继续按照策略 π 走，未来能得到的 期望回报。 • 直观理解：Q = Quality of taking an action in a state。

例子： 在骑士题里， Q(i,j,k,a)={从位置(i,j)，先走动作a，再走k−1步，存活的概率}Q(i,j,k,a) = \text\{从位置 (i,j)，先走动作 a，再走 k-1 步，存活的概率\}Q(i,j,k,a)={从位置(i,j)，先走动作a，再走k−1步，存活的概率}

奖励函数 R vs 价值函数 V (以 LeetCode 688 骑士题为例)

概念定义在骑士题里的体现特点
奖励函数 R环境即时反馈：从状态 (i,j) 执行动作 a 到 (i',j') 时给的 reward- 每走一步不发奖励- 只在最后一步结束时判断： - 若出界 → R = 0 - 若仍在棋盘 → R = 1- 短期的、即时的- 由环境定义，不依赖策略
价值函数 V在某个策略下，从状态 s 出发能得到的 长期累计 reward 的期望- V(i,j,k) = 从 (i,j) 出发，剩余 k 步，最终仍然留在棋盘上的概率- 长期的、期望的- 与策略 π 有关- 由 Bellman 方程递推计算
动作价值函数 Q在某个状态下执行某个动作 a 后，长期回报的期望- Q(i,j,k,a) = 从 (i,j) 出发，先走动作 a 到 (i',j')，再走 k-1 步存活的概率- 比 V 更细，考虑“状态+动作”组合

✅ 直观理解：

  • Reward R = 当下出界了吗？ → 0/1
  • Value V = 未来存活的概率（长期结果的期望）
  • Q = 未来存活概率，但先指定了第一步动作

2. 回报定义​

  • 回报 = 剩余 k 步走完后仍然在棋盘上的概率。
  • 即：
Gt=1 ⁣[{stillinsideafterallsteps}] G_t = \mathbf{1}\!\left[ \text\{still inside after all steps\} \right] Gt​=1[{stillinsideafterallsteps}]

3. 价值函数​

  • 定义 状态价值函数：
V(i,j,t)={从位置(i,j)出发，在t步后仍然在棋盘上的概率} V(i, j, t) = \text\{从位置 (i,j) 出发，在 t 步后仍然在棋盘上的概率\}V(i,j,t)={从位置(i,j)出发，在t步后仍然在棋盘上的概率}

4. Bellman 方程​

  • 边界条件：

    V(i,j,0)=1{if(i,j)在棋盘内,否则=0}V(i, j, 0) = 1 \quad \text\{if (i,j) 在棋盘内, 否则 = 0\}V(i,j,0)=1{if(i,j)在棋盘内,否则=0}

  • 递推关系：

    V(i,j,t)={1}{8}∑{{move}a∈A}V(i′,j′,t−1)V(i, j, t) = \frac\{1\}\{8\}\sum_\{\text\{move \} a \in A\} V(i', j', t-1)V(i,j,t)=1{​}{8}{∑​{move}a∈A}V(i′,j′,t−1)

其中 (i', j') 是从 (i, j) 出发做动作 a 的新位置。

5. 解题思路提示​

  1. 用 DP 或记忆化递归实现 V(i,j,k)。
  2. 基本情况：k == 0 时，检查是否还在棋盘内。
  3. 状态转移：枚举 8 个 knight moves，累加子状态概率。
  4. 最终答案 = V(row, col, K)。

6. 类比 Sutton & Barto 第三章​

  • 棋盘位置 + 剩余步数 = 机器人问题的 电池电量状态。
  • 骑士的 8 种走法 = 机器人动作 (search / wait)。
  • 出界 = 失败 = 电池耗尽 = -1 回报。
  • 递推概率 = Bellman equation。

所以这道题就是书里的 回收机器人 的一个变体。

没有memoization的答案​

"""
v(i, j, t)代表从现在（还没有行动）开始到最后还剩t步时成功的概率，


k = 2: 
v(i, j, t=0): 现在已经游戏结束，0 if outside, 1 if inside
v(i, j, t = 1) = 0 * 1/8 for those outside at t = 0, sum(1 * 1/8 for those inside at t = 0)
v(i, j, t = 2) =  0 * 1/8 for those outside at t = 1, sum(v(i, j, 1) * 1/8 ) for those inside at t = 1) 


注意： 每一轮都会有出去的，出去了以后，如果只是算 v(i, j, 2) = sum(v(i, j, 1) * 1/8 ) for those inside at t = 1)，会忽略那些出去了又进来的。 

所以一定要算v(i, j, 0) =  0 * 1/8 for those outside at t=1, sum(v(i, j, 1) * 1/8 ) for those inside at t=1)


最后返回v(i, j, 2)

"""
class Solution:
    def knightProbability(self, n: int, k: int, row: int, column: int) -> float:

        state_value = self.calc_V(row, column, k, n)
        return state_value
        
    
    def calc_action_set(self, i, j):
        xy_lst = [(-2, 1), (-1, 2), (1, 2), (2, 1), 
                (2, -1), (1, -2), (-1, -2), (-2, -1)]
        return [(i + x, j + y) for x, y in xy_lst]

    def in_bound(self, x, y, n):
        if 0 <= x < n and 0 <= y < n:
            return True
        else:
            return False

	
    def calc_V(self, i, j, t, n):
        if t == 0:
            return int(self.in_bound(i, j, n))
        state_value = 0
        for (r, c) in self.calc_action_set(i, j):
            if self.in_bound(r, c, n):
                next_state_value = self.calc_V(r, c, t - 1, n)
                state_value += next_state_value * 1/8

        return state_value

memoization​

举例：从中心出发走 2 步，很多路径都会在第 1 步落到同一格，且 steps_left=1 相同——这就是相同状态，应该共享答案。

在 calc_V 前面加

@functools.lru_cache(maxsize=None)

其实是存 以 i, j, t 为key的值

注意一定要写maxsize = None，不然就会超出lrucache缓存的范围

LeetCode 688 与强化学习 (RL) 的联系与转化

1. 原题回顾​

LeetCode 688: Knight Probability in Chessboard

  • 输入：棋盘大小 n，骑士起点 (r,c)，总共 K 步。
  • 骑士每步按国际象棋规则随机移动 8 种走法。
  • 问题：走完 K 步后，骑士仍在棋盘内的概率。

2. 抽象成 MDP​

状态 (S)

  • 可写为三元组 (s=(i,j,t))：当前坐标 ((i,j))，还剩 (t) 步。

动作 (A)

  • 8 种骑士走法。

转移概率 (P)

  • 原题中：每步等概率选择动作，即每个动作概率为 (1/8)。
  • 一旦越界 → 进入终止失败状态。

奖励 (R)

  • 终端奖励设为 1（成功存活）、0（失败出界）。
  • 中间步奖励=0。
  • 最终目标是期望奖励，也就是存活概率。

策略 (π)

  • 原题策略固定：每步等概率随机。
  • 如果允许选择动作，则变成控制问题：策略 π 可以优化目标。

3. Bellman 方程的应用​

策略评估 (已知固定随机策略)​

定义：

Vt(i,j)=Pr⁡(从 i,j 出发，剩 t 步仍存活)V_t(i,j) = \Pr(\text{从 i,j 出发，剩 t 步仍存活})Vt​(i,j)=Pr(从 i,j 出发，剩 t 步仍存活)

递推（Bellman 策略评估）：

Vt+1(i,j)=18∑a∈AVt(i′,j′)V_{t+1}(i,j) = \frac{1}{8}\sum_{a \in \mathcal{A}} V_t(i',j')Vt+1​(i,j)=81​a∈A∑​Vt​(i′,j′)
  • 如果 ((i',j')) 在棋盘内 → 值有效。
  • 如果越界 → 值为 0。
  • 边界：(V_0(i,j)=1)。

这就是题解里常见的 DP 公式。

4. 控制问题：需要选动作最大化目标​

如果动作不再固定随机，而是由智能体选择，就要解 Bellman 最优方程。

定义最优价值函数​

Vt\*(i,j)=max⁡πPr⁡(从(i,j)出发，剩 t 步后仍在棋盘内)V_t^\*(i,j) = \max_{\pi} \Pr(\text{从}(i,j)\text{出发，剩 }t\text{ 步后仍在棋盘内})Vt\*​(i,j)=πmax​Pr(从(i,j)出发，剩 t 步后仍在棋盘内)

Bellman 最优递推​

Vt+1\*(i,j)=max⁡a∈A(i,j)Vt\*(i′,j′)V_{t+1}^\*(i,j) = \max_{a \in \mathcal{A}(i,j)} V_t^\*(i',j')Vt+1\*​(i,j)=a∈A(i,j)max​Vt\*​(i′,j′)
  • 每一步不再取平均，而是取 最大值。
  • 最优策略 (\pi^*) 是：
πt+1\*(i,j)=arg⁡max⁡aVt\*(i′,j′)\pi_{t+1}^\*(i,j) = \arg\max_{a} V_t^\*(i',j')πt+1\*​(i,j)=argamax​Vt\*​(i′,j′)

5. 不同目标下的 Bellman 方程​

  1. 最大化存活概率

    Vt+1\*(i,j)=max⁡aVt\*(i′,j′)V_{t+1}^\*(i,j) = \max_{a} V_t^\*(i',j')Vt+1\*​(i,j)=amax​Vt\*​(i′,j′)

  2. 最大化存活步数（期望时间）
    每存活一步给奖励 1：

    Vt+1\*(i,j)=1+max⁡a Vt\*(i′,j′)V_{t+1}^\*(i,j) = 1 + \max_{a}\, V_t^\*(i',j')Vt+1\*​(i,j)=1+amax​Vt\*​(i′,j′)

  3. 折扣奖励（考虑未来不确定性）

    V\*(s)=max⁡a{r(s,a)+γ∑s′P(s′∣s,a)V\*(s′)}V^\*(s) = \max_a \{ r(s,a) + \gamma \sum_{s'} P(s'|s,a)V^\*(s') \}V\*(s)=amax​{r(s,a)+γs′∑​P(s′∣s,a)V\*(s′)}

6. 三类解法对照​

A. 有模型，有限时域​

  • 后向动态规划（Backward DP）：自底向上推，复杂度 (O(K\cdot n^2\cdot 8))。

B. 有模型，无限时域​

  • 值迭代 / 策略迭代，解折扣 MDP。

C. 无模型（只能采样）​

  • Q-learning / SARSA，利用 Bellman 目标做近似更新。

7. 小例子（伪代码）​

最优控制 DP​

def optimal_survival_prob(n, K, start_i, start_j):
    moves = [(2,1),(2,-1),(-2,1),(-2,-1),(1,2),(1,-2),(-1,2),(-1,-2)]
    V = [[[0.0]*n for _ in range(n)] for _ in range(K+1)]
    for i in range(n):
        for j in range(n):
            V[0][i][j] = 1.0  # base case

    for t in range(K):
        for i in range(n):
            for j in range(n):
                best = 0.0
                for di, dj in moves:
                    ni, nj = i+di, j+dj
                    if 0 <= ni < n and 0 <= nj < n:
                        best = max(best, V[t][ni][nj])
                V[t+1][i][j] = best

    return V[K][start_i][start_j]
